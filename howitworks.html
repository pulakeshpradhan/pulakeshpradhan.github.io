Google Earth Engine AI Assistant

Supercharging Your Geospatial Workflow with the Power of AI

A Technical Deep Dive

The Challenge: Complexity of Google Earth Engine (GEE)

  • Steep Learning Curve: GEE's powerful API can be daunting for new users.
  • Time-Consuming: Even experienced users spend significant time writing, debugging, and modifying scripts.
  • Error Prone: Small syntax errors can lead to frustrating debugging sessions.
  • Keeping Up: New datasets and API changes require continuous learning.

The Solution: A Seamless AI Experience

The Google Earth Engine AI Assistant is a Chrome extension that integrates the power of the Gemini API directly into the GEE Code Editor. This provides users with an AI-powered co-pilot for their geospatial analysis.

Core Features

Generate, Modify, and Fix GEE Code with Natural Language

  • Generate: Create new GEE scripts from a simple text prompt.
  • Modify: Edit existing code with natural language instructions.
  • Fix: Automatically correct errors in your scripts.
  • Run: Execute scripts directly from the side panel.

Target Audience

Who Can Benefit from This Tool?

  • Researchers: Accelerate their data analysis and experimentation.
  • Students: Learn GEE more effectively with an AI tutor.
  • Developers: Quickly prototype and build GEE applications.
  • GIS Analysts: Automate repetitive tasks and improve efficiency.
  • Anyone working with geospatial data in Google Earth Engine.

High-Level Architecture

The User Interface (UI)

The `sidepanel.html` and `sidepanel.css` Files

The user interacts with the extension through a clean and intuitive side panel. This panel is built with standard HTML and styled with CSS to provide a seamless user experience.

  • API Key Input: Securely stores the user's Gemini API key.
  • Action Buttons: "Run Script", "Fix Errors", "Modify", and "Generate".
  • Text Areas: For user prompts and modification instructions.
  • Status Bar: Provides real-time feedback to the user.

The Core Components

The Four Pillars of the Extension

  1. manifest.json: The "blueprint" that defines the extension's properties and permissions.
  2. background.js: The service worker that runs in the background and manages events.
  3. content.js: The script that is injected into the GEE webpage to enable communication.
  4. sidepanel.js: The "brains" of the operation, handling user interactions and API calls.

Deep Dive: `manifest.json`

The "Blueprint" of the Extension

The manifest.json file is a JSON-formatted file that provides essential information about the extension, such as its name, version, permissions, and the files it uses.

Key Manifest Properties

PropertyDescription
name, version, descriptionBasic information about the extension.
permissionsThe browser features the extension needs to access.
host_permissionsThe websites the extension is allowed to interact with.
actionDefines the extension's icon in the toolbar.
side_panelSpecifies the HTML file for the side panel.
backgroundDefines the service worker script.
content_scriptsLists the scripts to be injected into web pages.

Permissions Explained

  • storage: Allows the extension to store and retrieve the user's Gemini API key.
  • scripting: Enables the extension to inject and execute scripts on the GEE webpage.
  • sidePanel: Grants access to the Side Panel API.
  • tabs: Allows the extension to get information about the current tab.

Host Permissions

"https://code.earthengine.google.com/*"

This is a critical permission that allows the extension to interact with the Google Earth Engine Code Editor. Without it, the extension would not be able to read code, inject scripts, or automate actions on the page.

Deep Dive: `background.js`

The Service Worker

The background.js script is a service worker that runs in the background, independent of any webpage. It's an event-based script that listens for browser events and takes action accordingly.

Managing the Action Icon

The `manageActionIcon` Function

This function is responsible for enabling or disabling the extension's icon in the toolbar. The icon is enabled only when the user is on a Google Earth Engine Code Editor tab.

const manageActionIcon = async (tab) => {
    if (tab?.url?.startsWith("https://code.earthengine.google.com")) {
        await chrome.action.enable(tab.id);
    } else if (tab) {
        await chrome.action.disable(tab.id);
    }
};

Event Listeners in `background.js`

  • chrome.action.onClicked: Opens the side panel when the user clicks the extension's icon.
  • chrome.tabs.onUpdated: Checks the URL and enables/disables the action icon when a tab is updated.
  • chrome.tabs.onActivated: Ensures the action icon's state is correct when the user switches to a different tab.

Deep Dive: `sidepanel.js`

The Brains of the Operation

This is the most substantial script in the extension. It's responsible for:

  • Handling all user interactions within the side panel.
  • Managing the user's API key.
  • Constructing prompts for the Gemini API.
  • Making API calls to the Gemini service.
  • Injecting the returned code into the GEE editor.

Deep Dive: `content.js` and Script Injection

Bridging the Gap

The content.js script is a bridge between the extension's side panel and the GEE webpage. It's a small script that is injected into the GEE page and listens for messages from the extension.

The `injectScript` Function

This is a powerful utility function in sidepanel.js that allows us to execute arbitrary functions in the context of the main GEE webpage. This is how we are able to interact with the ACE editor and other elements on the page.

function injectScript(func, args = []) {
    return new Promise((resolve, reject) => {
        chrome.scripting.executeScript({
            target: { tabId: tabs[0].id },
            world: 'MAIN',
            func: func,
            args: args,
        }, (injectionResults) => { ... });
    });
}

Injected Functions

These are functions that are defined in sidepanel.js but are executed on the GEE webpage via injectScript.

  • getCodeAndErrors: Reads the current code from the ACE editor and scrapes any error messages from the GEE console.
  • injectCodeIntoEditor: Replaces or inserts the AI-generated code into the ACE editor.
  • clickGeeButton: Programmatically clicks buttons on the GEE page, such as the "Run" button.

The `world: 'MAIN'` Property

This is a crucial detail. By setting world: 'MAIN', we ensure that our injected scripts execute in the same JavaScript context as the GEE webpage itself. This is what allows us to access the ace editor instance and other page-specific variables.
